# 施工方案AI编写技能

**技能名称**: construction-plan-writer  
**版本**: 1.0.0  
**语言**: 中文  
**平台**: OpenClaw / Claude Code / 通用  
**标签**: `#施工方案` `#CAD分析` `#工程计算` `#中国规范`

---

## 概述

将甲方提供的CAD图纸、模板文档、技术要求，快速生成符合中国施工规范的专业施工方案。覆盖消防、钢结构、深冷空分、设备安装等类型。

## 适用场景

- 施工方/中介需要按甲方模板重写施工方案
- CAD图纸分析 + 数据提取（包括无文字层的炸开CAD）
- 吊车选型计算（臂长、负载率、钢丝绳校核）
- 按JGJ/GB规范套用标准章节结构
- 专业排版（封面/目录/表格/行距/字体）

## 前置能力

| 能力 | 说明 |
|------|------|
| PyMuPDF (fitz) | PDF文字层提取 + 无文字层转PNG |
| python-docx | Markdown→Word排版 + 封面/表格/图片 |
| 视觉模型 (minicpm-v) | CAD密集标注识别 |
| 工程计算 | 吊车验算 / 钢丝绳校核 / 地基承载力 |

## 输入

```
- 甲方模板文档（PDF/Word）
- CAD导出图纸（PDF，可能无文字层）
- 设备清单/报价单（Excel）
- 甲方要求（文字/语音/截图）
```

## 输出

```
- 完整施工方案 .docx（含封面、目录、正文、表格、计算书）
- 排版标准：A4/宋体12pt/1.5行距/灰底表头
```

## 工作流

### 步骤1：资料分析（30分钟）
1. 读取甲方聊天截图 → 提取核心诉求
2. 分析模板PDF → 提取目录结构
3. 分析CAD图纸：
   - 有文字层：`fitz.open().get_text()` 直接提取
   - 无文字层：`get_pixmap(dpi=120)` → PNG → 视觉模型识别
4. **校核模板 vs 图纸**：结构类型/尺寸/构件规格，以图纸为准

### 步骤2：方案编写（60分钟）
1. 按标准章节结构组织（10章）
2. 计算书：依据JGJ 276-2012，标注公式来源
3. 数据100%来自客户文件，不确定的写"待补充"

### 步骤3：排版交付（30分钟）
1. python-docx 生成 .docx
2. 自动插入附图（总平图/柱网图/剖面图）
3. 自检清单校验

## 标准章节结构

```
1. 工程概况
2. 编制依据
3. 施工计划
4. 施工工艺技术
5. 施工保证措施
6. 施工管理及人员配备
7. 验收要求
8. 应急处理措施
9. 计算书
10. 结论
```

## 核心规范

- JGJ 276-2012 建筑施工起重吊装工程安全技术规范
- GB 50205-2020 钢结构工程施工质量验收标准
- GB 50661-2011 钢结构焊接规范
- GB 8918-2006 重要用途钢丝绳
- JGJ 80-2016 建筑施工高处作业安全技术规范

## ⚠️ 必读踩坑清单

1. **数据100%来自客户文件** — 禁止从网上搜类似项目数据填充
2. **先验证结构类型** — 模板可能是另一个项目的，以CAD图纸为准
3. **编制单位留空** — 不要从模板中猜测
4. **CAD炸开文字层** — 视觉模型识别精度有限，精确数据标注"以图纸为准"
5. **吊车型号以检验报告为准** — 行驶证可能写不同型号
